= ADR 0002 — Haec design pass
:revdate: 2026-06-22

== Status
Accepted (design).

== Context
Haec is the first front-end of the trope-particularity pipeline: it writes and
type-checks programs and emits the language-neutral Trope IR consumed by
`trope-checker`. The calculus (`trope-checker/spec/calculus.adoc`) leaves the
surface syntax and elaboration to the front-end; this ADR records that pass.

== Decisions
[cols="1,3",options="header"]
|===
| ID | Decision
| Design before compiler (HC-5) | Pin the surface grammar + elaboration + round-tripping examples first; scaffold `compiler/` and `lsp/` as named future work. No parser/codegen this pass.
| Five atoms + sugar | The honest fragment is generated by `attenuate`/`predicate`/`drop`/`sever`/`fuse`; `preserve`/`project`/`collapse`/`detach` are sugar (calculus expansions). The deceptive trio is not writable (HC-3).
| Lowering correctness is Haec's burden (O2) | Every construct in `design/elaboration.adoc` carries a stated O2 obligation. The checker's soundness is relative to faithful lowering; Haec owns the proofs (deferred, named — not asserted).
| Vendored IR schema | `design/trope-ir.schema.json` is a vendored copy of the published IR contract so `just check` validates examples standalone. The language depends on the IR (correct direction); the checker depends on no language (HC-1).
| RSR placement | `design/` (grammar, elaboration, vendored schema), `examples/`, `compiler/`, `lsp/` added to `root-allow.txt` with justification. Code = MPL-2.0, prose docs = CC-BY-SA-4.0.
|===

== Consequences
Haec's meaning is fixed and machine-checkable (examples round-trip through the
checker) before any implementation exists. A reader can see, per construct, what it
lowers to and what Haec must still prove about that lowering.
